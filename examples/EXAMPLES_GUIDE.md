## Getting Started

### Prerequisites

1. CloudForge API running locally or deployed to AWS
2. Python 3.11+ installed
3. API key configured (see main README.md)

### Setup

```bash
# Clone the repository
git clone <repository-url>
cd cloudforge-bug-intelligence

# Install Python dependencies
cd backend
pip install -r requirements.txt

# Start the API
python -m uvicorn cloudforge.api.main:app --reload --port 8000

# In another terminal, navigate to examples
cd examples
```

## Basic Workflow Example

The `api_examples/basic_workflow.py` script demonstrates a complete workflow:

### Configuration

Edit the script to set your API key:

```python
API_BASE = "http://localhost:8000"

```

### Running the Example

```bash
cd api_examples
python basic_workflow.py
```

### What It Does

1. **Creates a workflow** for a specified repository
2. **Polls for completion** with exponential backoff
3. **Retrieves bugs** detected during the scan
4. **Retrieves fixes** generated by the system
5. **Exports results** to a JSON file

## CI/CD Integration

### GitHub Actions

Use the provided GitHub Actions workflow:

1. Copy `integration_examples/github_actions_example.yml` to `.github/workflows/bug-detection.yml`
2. Add secrets to your repository:
   - `CLOUDFORGE_API_KEY`: Your API key
   - `CLOUDFORGE_API_URL`: Your API URL (e.g., https://api.cloudforge.example.com)
3. Push to trigger the workflow

The workflow will:
- Run on every push to main/develop
- Run on every pull request
- Fail the build if critical bugs are found
- Upload results as artifacts

### GitLab CI

```yaml
# .gitlab-ci.yml
bug-detection:
  stage: test
  script:
    - pip install requests
    - python examples/integration_examples/gitlab_ci.py
  only:
    - main
    - merge_requests
```

## Troubleshooting

### API Connection Issues

**Problem**: Cannot connect to API

**Solution**:
```bash
# Check if API is running
curl http://localhost:8000/health

# Check API logs
cd backend
python -m uvicorn cloudforge.api.main:app --reload --log-level debug
```

### Authentication Errors

**Problem**: 401 Unauthorized or 403 Forbidden

**Solution**:
1. Verify API key is correct in your script
2. Check API key configuration in `backend/config.py`
3. Ensure `X-API-Key` header is included in requests

### Rate Limiting

**Problem**: 429 Too Many Requests

**Solution**:
```python
# Add retry logic with exponential backoff
import time

def make_request_with_retry(url, headers, max_retries=3):
    for attempt in range(max_retries):
        response = requests.get(url, headers=headers)
        
        if response.status_code == 429:
            wait_time = 2 ** attempt  # Exponential backoff
            print(f"Rate limited, waiting {wait_time}s...")
            time.sleep(wait_time)
            continue
        
        return response
    
    raise Exception("Max retries exceeded")
```

### Workflow Timeout

**Problem**: Workflow takes too long to complete

**Solution**:
1. Check workflow status in API logs
2. Verify AWS credentials are configured
3. Check CloudWatch logs for agent errors
4. Increase timeout in polling logic

### No Bugs Detected

**Problem**: Workflow completes but finds no bugs

**Solution**:
1. Verify repository contains code files
2. Check file patterns in scan_options
3. Review severity filter settings
4. Check if mock mode is enabled (should be disabled for real scans)

## Advanced Usage

### Custom Polling Strategy

```python
def smart_poll(workflow_id, initial_delay=5, max_delay=60):
    """Poll with adaptive delay based on workflow progress."""
    delay = initial_delay
    last_agent = None
    
    while True:
        response = requests.get(
            f"{API_BASE}/workflows/{workflow_id}",
            headers=HEADERS
        )
        
        workflow = response.json()
        current_agent = workflow.get("current_agent")
        
        # Increase delay if agent hasn't changed
        if current_agent == last_agent:
            delay = min(delay * 1.2, max_delay)
        else:
            delay = initial_delay
        
        last_agent = current_agent
        
        if workflow["status"] in ["completed", "failed"]:
            return workflow
        
        time.sleep(delay)
```

### Custom Result Processing

```python
def analyze_bug_trends(workflow_ids):
    """Analyze bug trends across multiple workflows."""
    all_bugs = []
    
    for workflow_id in workflow_ids:
        bugs = get_bugs(workflow_id)
        all_bugs.extend(bugs)
    
    # Group by file
    by_file = {}
    for bug in all_bugs:
        file_path = bug["file_path"]
        by_file.setdefault(file_path, []).append(bug)
    
    # Find hotspots
    hotspots = sorted(
        by_file.items(),
        key=lambda x: len(x[1]),
        reverse=True
    )
    
    print("Top 10 Bug Hotspots:")
    for file_path, bugs in hotspots[:10]:
        print(f"  {file_path}: {len(bugs)} bugs")
```
